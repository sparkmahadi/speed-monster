Some Questions related to JavaScript

  • What is the difference between Local Storage and Session Storage?

    Local Storage: Local Storage is a web storage method that helps us store data on the clientâ€™s computer in the form of key/value pairs in a web browser. The data is stored in local storage for a lifetime unless the user manually deletes it from the browser. It does not expire even when the user closes the window or tab. Instead, the data remains in the browser until and unless the browser's memory is cleared.
    Session Storage: The Session Storage is very similar to the Local Storage. Still, the main difference lies in the lifespan as it persists in the browser until its current tab or session is on. Once you close the tab or terminate the session, the data on session storage also gets lost.

  • What is the difference between Global Scope and Block Scope?

    Global Scope: If a variable is declared outside all functions or curly braces ({}), it exists in the global scope. That means, It can be accessed by any line of code in the program, including inside blocks. So, the global scope means the scope or area which is existed all over the code.
    Block Scope: Block Scope means the scope or area inside a block of codes like inside a function or conditionals or other curly braces ({}). In this case, the variable can be accessed only inside the scope. That means, the variable only exists inside the block and cannot be accessed from outside of the block.

  • How the JS Event Loop works?

    Event Loop in JS: The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process. Basically, it is a component of js which is one of the helper for JS to complete asynchronous tasks.

  • In how many ways, can we get 'undefined'?

    Ways of getting 'undefined':

    • A variable which is not initialized yet will give the value 'undefined'.
    • A function with no return.
    • A parameter that is not passed.
    • A return in a function which doesn't have any value beside it.
    • A property of an object which doesn't exist.
    • Trying to access an element of an array out of it's index range.
    • An element of array which is deleted.
    • Assigning a value as 'undefined'.
